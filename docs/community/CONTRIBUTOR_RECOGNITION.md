# Contributor Recognition Program

**Version:** 1.0  
**Date:** January 2026  
**Status:** Active

## Overview

The DotNetAgents Contributor Recognition Program celebrates and acknowledges the valuable contributions of our community members. We believe in recognizing all forms of contribution, from code to documentation to community support.

### ðŸŒ± New Contributor

**Criteria:**
- First pull request merged
- First issue reported and fixed
- First documentation improvement
- First helpful answer in community

**Recognition:**
- Welcome message in Discord
- Listed in release notes
- GitHub profile badge (optional)
- Thank you in contributor list

### ðŸŒ¿ Active Contributor

**Criteria:**
- 5+ merged pull requests
- Regular participation in discussions
- Helpful community member
- Consistent contributions over 3+ months

**Recognition:**
- Special role in Discord (@Active Contributor)
- Featured in quarterly newsletter
- Listed in "Active Contributors" section
- Invitation to contributor meetings

### ðŸŒ³ Core Contributor

**Criteria:**
- 20+ merged pull requests
- Significant feature contributions
- Maintainer trust and recommendation
- Mentoring other contributors
- 6+ months of consistent contribution

**Recognition:**
- Special role in Discord (@Core Contributor)
- Listed in README contributors section
- Invitation to maintainer meetings
- Voting rights on community decisions
- Featured contributor profile

### ðŸŒ² Maintainer

**Criteria:**
- Long-term commitment (1+ years)
- Project leadership
- Community stewardship
- Maintainer nomination and approval
- Significant impact on project direction

**Recognition:**
- Maintainer role with full permissions
- Listed as maintainer in README
- Decision-making authority
- Featured in project documentation
- Annual maintainer recognition

## Recognition Methods

### 1. GitHub Profile Badges

Contributors can display badges on their GitHub profiles:

- **New Contributor Badge**: First contribution
- **Active Contributor Badge**: 5+ contributions
- **Core Contributor Badge**: 20+ contributions
- **Maintainer Badge**: Maintainer status

**How to Get:**
- Automatically assigned based on contributions
- Can be manually requested via issue

### 2. Release Notes

All contributors are listed in release notes:

```markdown
## Contributors

Thank you to all contributors who made this release possible:

### New Contributors
- @username - First contribution

### Active Contributors
- @username - Multiple improvements

### Core Contributors
- @username - Major feature work
```

### 3. Community Showcase

Featured contributor profiles on:
- Community website
- Quarterly newsletter
- Social media highlights
- Conference presentations

### 4. Annual Contributor Awards

Annual recognition for:
- **Most Helpful Contributor**: Most community support
- **Best Feature Contribution**: Most impactful feature
- **Documentation Excellence**: Best documentation work
- **Rising Star**: Most promising new contributor
